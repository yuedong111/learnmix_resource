01 / Typography

标题与正文排版

标题的专业感通常来自字重、字距、行高、宽度控制，而不是只靠“换一个字体”。

原始效果 + CSS

Track every dollar with clarity

Everything is centered. The line-height is loose, the headline weight is generic, and the spacing feels like it was never intentionally tuned.

原始参数
.hero-title { font-size: 44px; font-weight: 700; letter-spacing: 0; line-height: 1.18; text-align: center; } .body-copy { font-size: 16px; line-height: 1.8; max-width: 100%; }
专业效果 + CSS

Track every dollar with clarity

更紧的标题字距、更克制的正文宽度、更明确的左对齐，会立刻提升信息层级和阅读节奏。

专业参数
.hero-title { font-size: 50px; font-weight: 650; letter-spacing: -0.05em; line-height: 0.97; text-align: left; } .body-copy { font-size: 15px; line-height: 1.65; max-width: 48ch; }
专业效果说明

关键属性：font-weight: 650、letter-spacing: -0.05em、line-height: 0.97 和 max-width: 11ch 让标题更紧、更有重心；15px / 1.65 的正文节奏也更稳。

是否属于实际开发用法：这是实际项目里非常常见的写法，尤其在 landing page、营销页和产品首页中，通常会和品牌字体、断点一起微调。

全置中 Hero vs 分栏 Hero

专业效果 + CSS

Simple hero section

是否属于实际开发用法：属于真实开发里的常见布局方式，一般会用 Grid 或 Flex 组合实现，并补上移动端断点。

03 / Border & Shadow

实色边框 + 阴影 vs 低透明度 outline

新手常见问题是边框太实、阴影太黑，叠在一起看起来发脏。

是否属于实际开发用法：这是很常见的卡片处理手法，但实际项目里通常会沉淀成 design token，而不是每张卡片单独硬编码。

是否属于实际开发用法：非常符合实际开发习惯，通常会做成 Button 组件的 size 和 variant，不会在页面里散落手写。

07 / Reading Width

铺满容器 vs ch 字符宽度控制

文案宽度不应只服从布局栅格，也要服从阅读体验。

原始效果 + CSS

这段正文直接铺满了可用空间，一旦屏幕变宽，用户每一行扫描的长度会明显变长，阅读负担会变重，也会让营销页面显得更松、更散。

原始参数
.text-block { max-width: 100%; line-height: 1.8; }
专业效果 + CSS

这段正文把宽度收敛在 40ch 左右，让一行文本控制在更舒服的阅读节奏里。

专业参数
.text-block { max-width: 40ch; line-height: 1.65; }
专业效果说明

关键属性：max-width: 40ch 搭配 line-height: 1.65，把每行字数压在舒适区间里，阅读节奏会马上变顺。

是否属于实际开发用法：这是正文排版里非常实用的真实做法，尤其适合介绍文案、说明文和摘要区。

仪表盘卡片的差别常常不在信息本身，而在标题层级、边界、数字强调和图表占比。

原始效果 + CSS
Revenue
$82,451
Expenses
$21,804
原始参数
.widget { border: 1px solid #d1d5db; border-radius: 14px; box-shadow: 0 10px 24px rgba(0,0,0,.12); }
专业效果 + CSS
Revenue
$82,451
+12.4% vs last month
Expenses
$21,804
-3.1% vs last month
专业参数
.widget { outline: 1px solid rgba(2,6,23,.08); border-radius: 18px; box-shadow: 0 14px 32px rgba(15,23,42,.08); } .metric { letter-spacing: -0.04em; }
专业效果说明

关键属性：widget 的 outline、18px 圆角、分层阴影，以及 metric 的负字距，让信息卡片更稳、更像仪表盘部件。

是否属于实际开发用法：属于非常常见的 dashboard 设计模式，真实开发里会再结合骨架屏、数据状态和图表组件。

21 / Auth

登录页

登录页不是只要输入框和按钮就结束，结构、层级、说明和品牌语气会直接影响专业感。

原始效果 + CSS
Sign in
原始参数
.auth-card { padding: 18px; border: 1px solid #d1d5db; border-radius: 14px; }
专业效果 + CSS
Welcome back
Sign in to Axiom

Use your workspace email to continue.

专业参数
.auth-card { padding: 22px; outline: 1px solid rgba(2,6,23,.08); border-radius: 22px; background: rgba(255,255,255,.96); }
专业效果说明

关键属性：22px 的卡片内边距、22px 圆角、半透明白底和浅 outline，让登录框既聚焦又不压迫；标题字距收紧也会更显精致。

是否属于实际开发用法：这是实际登录页常见的视觉写法，项目里一般会在 Auth layout 组件层处理容器和表单样式。

23 / Pagination

Pagination

分页控件不是“数字按钮列表”，而是浏览节奏控制器。专业版通常更轻、更紧凑。

原始效果 + CSS

专业效果 + CSS

专业参数
.page-item { padding: 8px 11px; border: none; border-radius: 12px; font-weight: 600; } .page-item.active { background: #111827; color: #fff; }

是否属于实际开发用法：这是实际产品里很常见的分页处理方式，项目中通常还会补禁用态、悬停态和小屏折叠逻辑。

Drawer / Slide-over

抽屉组件是“半层级容器”，边角、阴影和顶部 handle 会直接影响专业感。

原始效果 + CSS
Quick actions

Manage invoice status and reminders.

原始参数
.drawer { border: 1px solid #d1d5db; border-radius: 16px 16px 0 0; box-shadow: 0 -8px 24px rgba(0,0,0,.14); }
专业效果 + CSS
Quick actions
Manage invoice reminders

专业参数
.drawer { outline: 1px solid rgba(2,6,23,.08); border-radius: 24px 24px 0 0; box-shadow: 0 -18px 42px rgba(15,23,42,.16); } .handle { background: #dbeafe; }

是否属于实际开发用法：很符合真实产品做法，但生产环境里通常会额外处理动效、遮罩、焦点锁定和手势交互。

这份合并版最适合怎么用

  1. 先只看左右视觉差异，判断你第一眼感受到的“专业感”到底来自哪里。
  2. 再对照两边 CSS，确认差异是来自排版、边界、状态层级，还是信息密度控制。
  3. 最后阅读右侧新增说明，把这些细节判断转成可复用的开发规则，而不是停留在“感觉更高级”。
