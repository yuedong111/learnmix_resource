# 接口设计任务：设计 FAQ 草稿生成接口

## 任务目标
请设计一个“FAQ 草稿生成接口”，供知识库后台在文档解析完成后触发调用。

## 背景上下文
该接口服务于：
- 业务场景：从已解析文档中生成 FAQ 草稿
- 调用方：知识库管理后台、异步任务调度器
- 数据来源：文档解析结果、chunk 数据、LLM 生成结果
- 上下游依赖：文档服务、FAQ 审核模块

## 输入材料
请参考以下信息：
- 业务需求：支持指定文档生成 FAQ 草稿，并记录来源段落
- 当前已有接口：已有文档详情查询接口
- 输入输出示例：输入 document_id；输出 FAQ 草稿列表和状态
- 性能要求：单次请求在异步模式下立即返回任务 ID
- 权限要求：仅知识库管理员可调用

## 约束条件
请遵守以下要求：
- 优先设计成异步接口
- FAQ 草稿必须能追溯来源 chunk_id
- 状态要可查询
- 不要把审核通过和生成接口混在一起

## 输出格式
请按以下结构输出：
1. 接口职责
2. 请求方式与路径
3. 请求参数
4. 响应结构
5. 错误码设计
6. 兼容性与扩展建议
7. 示例请求与响应

## 验收标准
输出结果应满足：
- 前后端可以直接联调
- 能支持异步生成与状态查询
- 能支持来源追溯
